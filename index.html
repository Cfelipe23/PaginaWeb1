Arquitectura de Sistemas Monolíticos

Un sistema monolítico es una arquitectura de software tradicional donde toda la aplicación se construye como una única unidad indivisible. Características principales:

  • Estructura Compacta: Todos los componentes están interconectados y se despliegan juntos.
  • Desarrollo Inicial Rápido: Más sencillo de desarrollar al principio.
  • Desventajas:
    • Difícil de escalar
    • Mantenimiento complejo
    • Cambios requieren redespliegue completo
  • Ejemplo: Aplicaciones web tradicionales donde frontend, backend y base de datos están en un solo proyecto.

Arquitectura Cliente-Servidor

Modelo de comunicación que distribuye las tareas y recursos entre proveedores (servidores) y solicitantes (clientes). Características principales:

  • Separación de Responsabilidades: Cliente se enfoca en la interfaz, servidor en el procesamiento.
  • Tipos de Implementación:
    • Cliente ligero (solo interfaz)
    • Cliente grueso (con lógica de negocio)
    • Servidor web, servidor de base de datos
  • Ventajas:
    • Mayor escalabilidad
    • Mejor distribución de recursos
    • Facilita la actualización independiente
  • Ejemplo: Servicios web, aplicaciones con API REST, servicios en la nube.
